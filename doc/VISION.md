## 一、命名哲学

### 为什么叫 "ClawCompany"

```
Claw（单数）= OpenClaw，一只爪子，一个 agent，单打独斗。
Claws（复数）= ClawCompany，一群爪子，一个公司，协同自治。
Company = 不是工具，是公司。有组织、有层级、有预算、有目标。
```

**Claw** 这个意象来自龙虾/螃蟹——每只爪子形态不同、功能不同，
但它们服务于同一个生物体的目标。Chairman 是大钳，Worker 是小爪，
Secretary 是触角。各司其职，协同完成觅食（赚钱）的目标。

**品牌延伸：**
- ClawAPI → 供给能量的血液（API 是公司的血管）
- ClawCompany → 由 Claws 组成的公司实体
- ClawMart → 未来的公司模板市场（类似 Paperclip 的 ClipMart）

## 二、WEB4.0 定义

### 每一代 Web 的核心突破

```
WEB 1.0 — Read
  人类创建内容，人类阅读内容。
  突破：信息可以在全球流通。
  人类角色：生产者 + 消费者

WEB 2.0 — Read + Write
  人类在平台上创建和互动。
  突破：人人都是内容创作者。
  人类角色：创作者 + 互动者（但平台拥有一切）

WEB 3.0 — Read + Write + Own
  区块链让用户拥有自己的数据、身份、资产。
  突破：所有权回归个人。
  人类角色：所有者 + 治理者（但仍然亲自执行所有工作）

WEB 4.0 — Read + Write + Own + Delegate
  AI agents 自治执行。人类设定目标，Claws 完成一切。
  突破：执行权委托给 AI，人类只保留决策权。
  人类角色：目标设定者 + 董事会
```

### WEB4.0 的核心等式

```
WEB4.0 = 人类意图 + AI 自治执行 + 加密经济激励

即：
- 人类设定目标（意图层）
- AI agents 自主分解、委派、执行（执行层）
- 加密货币/链上支付自动结算（激励层）
```

ClawCompany 天然契合这三层：
1. **意图层** — 用户告诉公司一个目标
2. **执行层** — Chairman 分解，CEO 规划，CTO 实现，Worker 执行
3. **激励层** — ClawAPI 用 USDC/USDT 结算，加密原生

## 三、产品哲学

### 核心信条：人类只做一件事 — 设定目标

```
❌ 传统方式：
   人类 → 打开 ChatGPT → 写 prompt → 等结果 → 不满意 → 改 prompt → 再等 → 手动整合
   人类参与每一步。每一个 AI 交互都需要人类在场。

❌ OpenClaw 方式：
   人类 → 启动 OpenClaw → 选模型 → 给指令 → agent 执行 → 遇到问题 → 人类介入
   更好了，但还是单 agent，遇到复杂任务人类还是要拆分和协调。

✅ ClawCompany 方式：
   人类 → "帮我分析 DeFi 市场并写一份报告"
   Chairman: "好，我来规划。"
     → CEO: "我把它拆成 3 个子任务。"
       → Researcher: "我去搜集数据。"
       → Worker: "我整理格式。"
     → CEO: "数据收到，让我审核... 有两处需要补充。"
       → Researcher: "补充完毕。"
     → Secretary: "报告定稿，发送给用户。"
   人类：（喝着咖啡看完报告）"批准" or "修改方向再来一次"
```

### The Board of Directors 模型

```
在 ClawCompany 里，人类 = 董事会（Board of Directors）

董事会做什么：
  ✓ 设定公司愿景和使命
  ✓ 批准重大决策
  ✓ 审查财务报告（成本追踪）
  ✓ 雇佣/解雇高管（增删改角色）
  ✓ 否决权（随时暂停/覆盖任何 agent）

董事会不做什么：
  ✗ 不亲自写代码
  ✗ 不亲自做研究
  ✗ 不亲自排版文档
  ✗ 不协调 agent 之间的工作
  ✗ 不担心用哪个模型

这就是 WEB4.0 的工作方式：
人类掌握方向盘，Claws 踩油门。
```

```
┌─────────────┬────────────────────┬───────────────────┐
│ 维度         │ Paperclip          │ ClawCompany       │
├─────────────┼────────────────────┼───────────────────┤
│ 口号         │ Zero-human company │ Build for WEB4.0  │
│ 人类角色     │ 配置 + 监控        │ 只设目标           │
│ Agent 来源   │ 外接               │ 内置 + 可外接      │
│ 模型策略     │ Agent 自带         │ 角色绑模型，自动路由│
```

Paperclip 说 "zero-human company"，但它本质是一个项目管理器——
你还是要配置每个 agent、设计 heartbeat、手动分配任务。

ClawCompany 说 "Claws Autonomous"——
你告诉目标，Claws 自己组织、自己分工、自己执行。
你随时可以介入，但你不必须介入。

## 五、用户故事

### 故事 1：独立创业者

```
Alex 想做一个 DeFi 数据分析网站。

Day 1:
  $ npx clawcompany init
  $ clawcompany goal "Build a DeFi TVL dashboard website"

  Chairman (Opus) 收到目标，制定战略：
    1. Research top DeFi protocols
    2. Design data architecture
    3. Build frontend
    4. Deploy

  CEO (Sonnet) 拆解为 12 个任务，分配给团队。
  CTO (GPT-5.4) 设计技术方案，选 Next.js + Python backend。
  Workers (Flash-Lite) 并行抓取 DeFi Llama 数据。

Day 2:
  Alex 醒来打开 Dashboard：
  - 8/12 任务完成
  - 花了 $1.47
  - CTO 遇到 API rate limit，已自动切换备用数据源
  - 一个前端组件待 Alex 审批（涉及设计决策）

  Alex 审批了设计，喝咖啡。

Day 3:
  网站上线。总成本 $3.82。
  Alex 只做了两件事：设定目标 + 审批一个设计决策。
```

### 故事 2：交易团队

```
用户使用 Trading Desk 模板：

  $ clawcompany init --template trading-desk
  $ clawcompany goal "Monitor ETH/BTC ratio, buy ETH when ratio < 0.04"

  Market Analyst (Gemini Pro) 每 5 分钟检查价格。
  Data Collector (Flash-Lite) 从 3 个 DEX 抓实时数据。
  Execution Trader (Sonnet) 在条件触发时构造交易。
  Chairman (Opus) 审批大额交易（>$1000 的需要人类确认）。

  用户睡觉时，Claws 在工作。
  用户醒来时，交易已执行，报告已生成。
```

### 故事 3：内容公司

```
  $ clawcompany init --template content-agency
  $ clawcompany goal "每周发布 3 篇关于 AI 趋势的博客文章"

  Content Writer (Sonnet) 每周一生成 3 篇草稿。
  SEO Specialist (GPT-5 mini) 优化标题和关键词。
  Editor (Opus) 最终审校。
  Publisher (Flash-Lite) 格式化并发布到 CMS。

  用户每周花 10 分钟浏览 Dashboard，
  批准或微调一两个方向。
```

## 七、README.md 草稿

```markdown
# ClawCompany 🦞

> **Build for WEB4.0, Claws Autonomous.**
> Set the goal. Claws do the rest.

ClawCompany turns one API key into a full AI company.
Chairman thinks. CEO plans. CTO codes. Workers hustle.
You set the goal. They figure out the rest.

## What is WEB4.0?

WEB 1.0 = Read. WEB 2.0 = Read + Write. WEB 3.0 = Read + Write + Own.
**WEB 4.0 = Read + Write + Own + Delegate.**

In WEB4.0, you own an AI company that runs autonomously.
You're the board of directors. Claws are your team.

## Quick start

\`\`\`bash
npx clawcompany init
# Enter your ClawAPI key → company created → start working
\`\`\`

## How it works

1. **Set a goal** — "Build a DeFi analytics dashboard"
2. **Claws decompose** — Chairman breaks it into strategy, CEO into tasks
3. **Claws execute** — CTO codes, Workers fetch data, Secretary writes docs
4. **Claws deliver** — You review the result. Approve, adjust, or redirect.

## Default team (1 key, 8 models)

| Role | Model | What they do |
|------|-------|--------------|
| Chairman | claude-opus-4-6 | Strategic decisions |
| CEO | claude-sonnet-4-6 | Daily management |
| CTO | gpt-5.4 | Technical work |
| Secretary | gpt-5-mini | Briefings, summaries |
| Worker | gemini-flash-lite | Routine tasks |
| Fallback | gpt-oss-120b/20b | Low-balance survival |

Every role is fully customizable. Add, remove, rename,
swap models, mix providers. Your company, your rules.

## Why ClawCompany?

- **One key, full company** — ClawAPI gives you 8 models across 4 providers
- **Right model for each job** — Opus decides, Flash-Lite hustles
- **50%+ cost savings** — Tasks cascade down to cheaper models
- **Fully customizable** — Add roles, swap models, mix providers
- **WEB4.0 native** — Crypto payments, agent autonomy, human governance

- [ClawAPI](https://clawapi.org) — The blood of the company
- [OpenClaw](https://github.com/anthropics/openclaw) — Where the Claw was born
```
