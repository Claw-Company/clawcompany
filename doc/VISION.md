# ClawCompany — Vision & Brand

> **Build for WEB4.0, Claws Autonomous.**
> The only thing humans need to do is tell the Claws what goal to achieve.

---

## 1. Naming philosophy

```
Claw (singular)  = OpenClaw — one claw, one agent, working alone.
Claws (plural)   = ClawCompany — a pack of claws, a company, autonomous collaboration.
Company          = Not a tool. A company — with org structure, hierarchy, budgets, and goals.
```

The **Claw** evokes lobster/crab anatomy — each claw differs in form and function, but they serve the same organism's goals. Chairman is the big claw, Worker is the small one, Secretary is the antenna. Each plays its role to feed (earn money) together.

**Brand extensions:**
- ClawAPI → The blood supply (API is the company's circulatory system)
- ClawCompany → The corporate entity made of Claws
- ClawMart → Future company template marketplace

## 2. WEB4.0 definition

| Era | Capability | Human role |
|-----|-----------|------------|
| WEB 1.0 | Read | Producer + consumer |
| WEB 2.0 | Read + Write | Creator + participant (platforms own everything) |
| WEB 3.0 | Read + Write + Own | Owner + governor (still executes all work) |
| **WEB 4.0** | **Read + Write + Own + Delegate** | **Goal setter + board of directors** |

### The WEB4.0 equation

```
WEB4.0 = Human Intent + AI Autonomous Execution + Crypto-Economic Incentives
```

ClawCompany fits all three layers:
1. **Intent layer** — User tells the company a goal
2. **Execution layer** — Chairman decomposes, CEO plans, CTO builds, Workers execute
3. **Incentive layer** — ClawAPI settles in USDC/USDT, crypto-native

---

## 3. Product philosophy: the Board of Directors model

In ClawCompany, the human is the **Board of Directors**.

What the board does:
- Set company vision and mission
- Approve major decisions
- Review financials (cost tracking)
- Hire/fire executives (add/remove/edit roles)
- Veto power (pause/override any agent at any time)

What the board does NOT do:
- Write code, conduct research, format documents
- Coordinate work between agents
- Worry about which model to use

**Humans hold the steering wheel. Claws step on the gas.**

## 5. Brand voice

```
Primary color:   Coral (#D85A30) — lobster/crab, from ClawAPI branding
Secondary color: Teal (#1D9E75) — ocean, complementary
Neutral:         Gray (#888780)
Accent:          Amber (#BA7517) — budget/cost elements
```

**Copy rules:**
- Concise and punchy: "Set the goal. Claws do the rest."
- No corporate jargon: never "synergy", "leverage", "AI-powered solution"
- Use analogies: "If OpenClaw is an employee, ClawCompany is the company."
- Technical but not show-off: "8 models, 4 providers, 1 API key."

**Taglines:**
- Primary: "Build for WEB4.0, Claws Autonomous."
- Secondary: "Set goals, not prompts." / "Not one agent. A whole company."

---

## 6. Ecosystem positioning

```
ClawCompany        — Autonomous company layer (goal in, result out)
    ↓
ClawAPI            — Model supply layer (8 models, 1 key, crypto-native)
    ↓
OpenClaw / Claude Code / Codex — Agent tools layer (single-agent execution)
    ↓
Foundation Models  — Raw inference (Opus, Sonnet, GPT, Gemini, OSS)
```

ClawCompany is the top layer. It doesn't replace any layer below — it combines them into an autonomous entity.
